## Project Description

Collections Intelligence Agent is a business-focused multi-agent system for
accounts receivable teams. It helps a finance user ask natural-language
questions about customer balances, AR aging, overdue exposure, invoices, and
collection follow-ups while keeping sensitive data handling and human approval
enforced in code.

The system is built with Google ADK and uses three agents:

- The Orchestrator agent routes requests, coordinates specialist agents, and
  owns the draft approval workflow.
- The Financial Analyst agent answers questions about balances, invoices,
  aging buckets, overdue accounts, and exposure risk.
- The Communications agent drafts professional, tier-aware collection notices
  and saves them as internal drafts for review.

The agents do not connect directly to the database. All data access goes
through a custom MCP server backed by SQLite. This MCP layer exposes tools for
customer summaries, invoices, AR aging, overdue account ranking, draft creation,
draft listing, and draft approval.

Safety is a central part of the design. All customer, invoice, payment, tax,
and bank data is synthetic and generated by `scripts/seed_data.py` using Faker
with a fixed random seed. Customer `tax_id` and `bank_account_number` fields
are masked inside the MCP server before tool results can reach any model
context. The communications agent has no email, SMTP, notification, or sending
tool. It can only save drafts with `pending_review` status. Approval is an
explicit human action handled by the orchestrator and only changes the internal
draft status to `approved`; it does not send or enqueue a message.

The demo flow shows the user launching ADK Web, asking for the top overdue
accounts, drafting a collection notice for the highest-risk account, listing
the saved draft, and approving it through the human-in-the-loop gate. The repo
also includes a Dockerfile for a reproducible local demo, README documentation,
and architecture and cover assets for the final submission.

This project demonstrates the Agents for Business track by combining multi-agent
coordination, MCP-based tool access, deterministic synthetic financial data,
server-side PII masking, and a safe draft-only communications workflow.
