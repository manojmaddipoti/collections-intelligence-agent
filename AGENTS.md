## Project

Collections intelligence agent — a multi-agent system built for Kaggle's
"AI Agents: Intensive Vibe Coding" capstone (Agents for Business track).

Full problem statement and architecture: see README.md.

## Non-negotiable constraints

- **No real data.** All customer, invoice, and payment data must be
  synthetic, generated by `scripts/seed_data.py`. Never import or reference
  real company data of any kind.
- **No secrets in code or git history.** API keys live in `.env`
  (gitignored). Use `.env.example` as the template. Never hardcode a key,
  even temporarily, even in a comment.
- **PII masking is server-side, not prompt-side.** `tax_id` and
  `bank_account_number` must never leave the MCP server unmasked. Do not
  rely on instructing the LLM to "not mention" these fields — strip or
  redact them inside the tool function itself, before the result is
  returned to the agent.
- **The communications agent drafts only.** It must never be given a tool
  capable of actually sending an email or notice. Human approval is a hard
  gate enforced in code (e.g. a status field that starts as
  `pending_review` and can only flip to `approved` via an explicit human
  action). Approval is internal status only; it must not send, enqueue, or
  hand off a message to another system.

## Architecture

Orchestrator agent (ADK) -> Financial analyst agent + Communications agent
-> MCP server (PII masking happens here) -> SQLite (`data/ar_finance.db`)

Full diagram and rationale: see README.md once the agents are built.

## Tech stack

- Agent framework: Google ADK (Python)
- Tool/data layer: custom MCP server (Python `mcp` SDK)
- Database: SQLite, synthetic data via `Faker`
- LLM: Gemini, via Google AI Studio API key for development
- Packaging: Docker
- Interface: ADK CLI (`adk run` / `adk web`)

## Commands

```bash
# Setup
python3 -m venv .venv && source .venv/bin/activate
python -m pip install -r requirements.txt
cp .env.example .env   # then add your GOOGLE_API_KEY

# After activation, `python` resolves to `.venv/bin/python`.
# Regenerate synthetic data (idempotent, fixed seed -> same output every run)
python scripts/seed_data.py

# Run the MCP server standalone (once built)
python mcp_server/server.py

# Run the agent via the ADK CLI (once built)
adk run agents/

# Run the Dockerized ADK Web demo
docker build -t collections-intelligence-agent .
docker run --rm -p 8000:8000 --env-file .env collections-intelligence-agent
```

## Conventions

- Python: type hints on function signatures, a docstring on every public
  function, no bare `except:`.
- SQL inside tool functions: always parameterized queries, never
  string-interpolated input.
- Favor straightforward code over clever code — this repo doubles as a
  portfolio/interview artifact and a Kaggle submission judged partly on
  documentation quality.
- Every new MCP tool needs: a docstring describing what it does, an entry
  in README's tool table, and a masking review if it touches `CUSTOMERS`
  data.
